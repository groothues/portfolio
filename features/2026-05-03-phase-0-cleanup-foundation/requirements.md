# Requirements — Phase 0: Cleanup & Shared Foundation

## Context

The project skeleton was generated by Spring Initializr and contains one stub controller (`HomePageController`) placed in the old package layout (`api/page/`). Before any feature work begins, the package structure must match the SKILL.md standard and the application needs a shared HTML layout that all subsequent phases can reuse.

This phase produces no user-visible features — its output is a clean foundation that every later phase depends on.

## Scope

### In scope

- **Package rename**: move `HomePageController` from `de.groothues.portfolio.api.page` → `de.groothues.portfolio.ui.controller` to align with the SKILL.md layout.
- **CLAUDE.md update**: replace the old package layout table with the new structure (`ui/controller`, `api/controllers`, `domain/service`, etc.).
- **Base layout template**: `templates/layout/main.html` — wraps every page with a consistent `<head>`, navigation bar, and footer slot.
- **Fragment library**: `templates/layout/fragments.html` — reusable Thymeleaf fragments: `head`, `nav`, `flash-messages`.
- **Vendored HTMX**: `htmx.min.js` downloaded and committed to `static/vendor/htmx/` so the app has no CDN dependency in production.
- **Minimal CSS**: `static/css/app.css` using **Pico CSS** (classless semantic framework) loaded via `<link>` in the layout head fragment. Custom overrides only as needed.
- **Home page wired to layout**: `templates/home.html` updated to extend `layout/main.html`.

### Out of scope

- Spring Security / authentication.
- Any new page beyond the home page.
- Any Flyway migration or JPA entity.
- Real navigation targets — nav links for Dashboard, Portfolios, Instruments, Transactions are rendered as placeholders (`href="#"` or disabled) until those pages exist.

## Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| CSS framework | Pico CSS | Classless — one `<link>` gives clean typography and form styles with no class noise in templates |
| HTMX delivery | Vendored | Reproducible builds; no CDN dependency at runtime |
| Nav structure | Dashboard, Portfolios, Instruments, Transactions | Matches the four main sections in architecture.md §7 |
| Flash messages | Thymeleaf fragment in layout | Every page can show success/error messages without duplicating markup |
